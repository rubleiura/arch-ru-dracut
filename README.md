# 🐧 Упрощённая установка Arch Linux: Макет блочной установки

> 📘 **Философия проекта:** Архитектура «копируй-понимай-выполняй». Вместо слепого следования wiki, вы получаете готовую структуру команд с подробными комментариями, которую можно визуально редактировать, анализировать и безопасно запускать по блокам.

---

## 🛑 КРИТИЧЕСКОЕ ПРЕДУПРЕЖДЕНИЕ О БЕЗОПАСНОСТИ
⚠️ **Все макеты выполняют ПОЛНУЮ ОЧИСТКУ ВЫБРАННОГО ДИСКА** (`wipefs --all`, `sgdisk -Z`).  
🔒 Перед началом **обязательно создайте резервную копию** всех важных данных на внешний носитель или облако.  
🔍 Всегда дважды проверяйте переменную `sdx` (имя диска) перед выполнением блоков разметки.  
📜 Автор не несёт ответственности за потерю данных в результате неверного следования инструкции.

## 🔄 Два способа выполнения установки

Проект поддерживает два рабочих сценария. Выберите тот, который лучше подходит под ваше оборудование и уровень комфорта.

| Критерий | 🌐 Способ 1: Удалённо через SSH | 💻 Способ 2: Локально в графической Live-среде |
|:---|:---|:---|
| **Оборудование** | 2 компьютера (управляющий + целевой) | 1 компьютер |
| **Загрузочный носитель** | Официальный Arch Linux Live ISO | EndeavourOS, Garuda Linux или любой Arch-based ISO с GUI |
| **Редактирование макета** | В графическом редакторе управляющего ПК | Встроенный редактор Live-среды (`mousepad`, `kate`, `gedit`) |
| **Удобство для новичков** | ⚙️ Требует базовых знаний сети/SSH | ✅ **Рекомендуется**: всё в одном окне, нет настройки сети |
| **Когда выбирать** | Установка на сервер, минималистичный ПК или несколько машин | Классическая установка на домашний ПК/ноутбук |

<details>
<summary>📋 <b>Инструкция по подготовке среды (нажмите, чтобы раскрыть)</b></summary>

### 🌐 Способ 1: SSH (2 компьютера)
1. Загрузите **целевой** компьютер в Arch Linux Live.
2. В терминале Live-среды выполните:
   ```bash
   pacman -Sy --noconfirm openssh && systemctl start sshd
   passwd root
   ip a  # запишите IP-адрес
   ```
3. На **управляющем** компьютере: `ssh root@<IP_целевой>`
4. Склонируйте репозиторий: `git clone https://github.com/rubleiura/arch-ru-dracut && cd arch-ru-dracut`
5. Откройте `.m` файлы в редакторе на управляющем ПК, выполните замену переменных и копируйте блоки в SSH-сессию.

### 💻 Способ 2: Локальный (1 компьютер)
1. Запишите образ **EndeavourOS** или **Garuda Linux** на USB (Ventoy/BalenaEtcher).
2. Загрузите компьютер с флешки в графический режим.
3. Откройте терминал и выполните:
   ```bash
   git clone https://github.com/rubleiura/arch-ru-dracut && cd arch-ru-dracut
   ```
4. Откройте нужный `.m` файл через `mousepad`, `kate` или `gedit`.
5. Разместите окна: слева — редактор, справа — терминал. Копируйте блоки команд мышью и вставляйте в терминал.
</details>

## 🧭 Выбор макета установки

| Файл макета | Тип прошивки | Файловая система | Шифрование (LUKS) | Логические тома (LVM) | 💡 Рекомендация |
|:---|:---|:---|:---|:---|:---|
| `Arch UEFI DRACUT.m` | UEFI (GPT) | BTRFS | ❌ | ❌ | **Лучший старт для новичков**. Современное оборудование, быстрая установка. |
| `Arch BIOS DRACUT.m` | BIOS/Legacy (GPT) | BTRFS | ❌ | ❌ | Старые ПК, совместимость с классическим загрузчиком. Требует раздел BIOS Boot. |
| `BTRFS LVM LUKS UEFI DRACUT.m` | UEFI (GPT) | BTRFS | ✅ | ✅ | Максимальная безопасность. Пароль при каждой загрузке. Гибкое управление разделами. |
| `BTRFS LVM LUKS BIOS DRACUT.m` | BIOS/Legacy (GPT) | BTRFS | ✅ | ✅ | Безопасность на legacy-железе. Требует отдельный незашифрованный `/boot`. |
| `Настройка после установки Arch Linux.m` | Любой | Любой | Любой | Любой | 📦 **Обязателен**. Выполняется ПОСЛЕ первой загрузки в новую систему. |

> 💡 **Совет для начинающих:** Если вы не уверены, какой тип загрузчика у вас, начните с `Arch UEFI DRACUT.m` или `Arch BIOS DRACUT.m`. Макеты с `LUKS` сложнее в восстановлении и требуют запоминания пароля.

## 📋 Пошаговая настройка переменных (КРИТИЧЕСКИ ВАЖНО)

В макетах используется **трёхэтапная замена переменных**. Не пытайтесь заменить всё сразу! Следуйте строгой последовательности, чтобы избежать ошибок разметки.

### 🔹 ЭТАП 1: `[EDIT] БЛОК 1` — Локализация зеркал
*Выполняется ПЕРЕД любыми действиями.*
1. Запустите `🔍 [LIVE] БЛОК 0` (скрипт определит ближайшие страны).
2. Скопируйте полученный список стран (например, `Russia,Estonia,Finland...`).
3. В `БЛОК 1` замените значение `Russia` на ваш список:
   ```
   Местоположение пользователя # Russia,Estonia,Finland,Latvia,Lithuania
   ```

### 🔹 ЭТАП 2: `[EDIT] БЛОК 4` — Основные переменные системы
*Выполняется ДО разметки диска.*
Используйте функцию **«Найти и заменить везде»** в графическом редакторе:
| Переменная | Пример значения | Что это значит? | ⚠️ Важно |
|:---|:---|:---|:---|
| `sdx` | `nvme0n1` или `sda` | Имя **физического** диска | Узнайте через `lsblk`. Без цифр! |
| `forename` | `yuri` | Логин пользователя | Только латиница, без пробелов |
| `User Name` | `Yuri Rublev` | Полное имя | Можно кириллицу, с пробелами |
| `Sony` | `arch-pc` | Имя компьютера (Hostname) | Без пробелов |
| `amd-ucode` | `intel-ucode` | Микрокод CPU | Замените только если у вас Intel |
| `defaults` | `ssd,noatime...` | Параметры монтирования BTRFS | Возьмите рекомендации из Блока 3 |
| `4G` | `8G` или `16G` | Размер раздела подкачки | Для гибернации `≥ объёма RAM` |

### 🔹 ЭТАП 3: `[EDIT] БЛОК 6` — Точные имена разделов
*Выполняется СТРОГО ПОСЛЕ выполнения `БЛОК 5` (разметки)!*
1. Скопируйте и выполните `💾 [LIVE] БЛОК 5`. Он создаст разделы и выведет `lsblk`.
2. Посмотрите на реальные имена разделов в терминале (например, `nvme0n1p1`, `nvme0n1p2`).
3. Вернитесь в редактор и в `БЛОК 6` замените шаблоны:
   - **Обычные макеты:** `sda1` → ESP/BIOS Boot, `sda2` → Root, `sda3` → Swap
   - **LUKS UEFI:** `sda1` → ESP, `sda2` → LUKS/LVM
   - **LUKS BIOS:** `sda1` → BIOS Boot, `sda2` → `/boot`, `sda3` → LUKS/LVM
> 📌 **Правило безопасности:** Не заменяйте `sda1/sda2/sda3` до выполнения Блока 5! Ошибка здесь приведёт к форматированию несуществующих разделов и остановке установки.

## 🚀 Пошаговый процесс установки

После корректной замены переменных в Блоках 1, 4 и 6, выполняйте макет строго по порядку:

| Этап | Название блока | Где выполняется | Что делает |
|:---|:---|:---|:---|
| 0️⃣ | `[LIVE] БЛОК 0` | Live-среда | Определяет географию для зеркал |
| 1️⃣ | `[EDIT] БЛОК 1` | Редактор | **Замена страны** (см. выше) |
| 2️⃣ | `[LIVE] БЛОК 2` | Live-среда | Подготовка среды, локаль, pacman, reflector |
| 3️⃣ | `[LIVE] БЛОК 3` | Live-среда | Диагностика оборудования, вывод `lsblk` |
| 4️⃣ | `[EDIT] БЛОК 4` | Редактор | **Замена основных переменных** (см. выше) |
| 5️⃣ | `[LIVE] БЛОК 5` | Live-среда | Создание разделов (`sgdisk`), вывод `lsblk` |
| 6️⃣ | `[EDIT] БЛОК 6` | Редактор | **Замена имён разделов** (см. выше) |
| 7️⃣ | `[LIVE] БЛОК 7` | Live-среда | Форматирование, LUKS/LVM, подтомы BTRFS, монтирование |
| 8️⃣ | `[LIVE] БЛОК 8` | Live-среда | `pacstrap`, генерация `fstab`, вход в `chroot` |
| 9️⃣–🔴 | `[CHROOT] БЛОК 9–16` | Внутри `chroot` | Настройка системы, ядро, GRUB, dracut, драйверы, DE |
| 🏁 | `[LIVE] БЛОК 17` | Live-среда | Размонтирование, `poweroff`, перезагрузка |

💡 **Совет по копированию:** Копируйте команды целиком блоками. Дожидайтесь завершения каждого блока перед переходом к следующему. Если терминал выдал ошибку — остановитесь, проанализируйте вывод, устраните причину.

## 🖥️ Графика, драйверы и пост-настройка

### 🔹 Видеодрайверы (БЛОК 14)
Находятся в `БЛОКЕ 14`. Выберите **только один** сценарий (Intel/AMD/NVIDIA). Для NVIDIA критически важно выполнить настройку `dracut` и параметр `nvidia-drm.modeset=1`, иначе Wayland и гибернация не заработают.

### 🔹 Окружения рабочего стола (БЛОК 16)
В `БЛОКЕ 16` представлены готовые команды для KDE Plasma, GNOME, XFCE4, MATE, Cinnamon, LXQt, LXDE. Выберите одну среду, скопируйте блок, вставьте в терминал (находясь в `chroot`). После завершения блока произойдёт автоматический выход `exit` из `chroot`.

### 🔹 Почему пост-настройка вынесена в отдельный файл?
Файл `Настройка после установки Arch Linux.m` выполняется **строго после перезагрузки** в новую систему. Он содержит:
- Настройку `yay`, `snapper`, `btrfs-assistant` (снапшоты и откаты)
- UFW, PipeWire, EasyEffects, ZRAM, TLP
- Приложения, темы, игровые обёртки (Lux-Wine), виртуализацию
Это гарантирует, что все пользовательские сервисы и AUR-пакеты устанавливаются в уже работающую среду, а не в `chroot`.

## 🆘 Отладка и обучение

| Симптом | Вероятная причина | Решение |
|:---|:---|:---|
| Чёрный экран / зависание при загрузке | Отсутствуют параметры `rd.luks`, `rd.lvm` или не собран initramfs | Проверьте `БЛОК 12`. Выполните `dracut --force` и `grub-mkconfig`. |
| GRUB не видит систему | Ошибка в `--efi-directory` или отсутствии `part_gpt` | UEFI: `grub-install --target=x86_64-efi --efi-directory=/boot`. BIOS: раздел `EF02` обязателен. |
| `unrecognized filesystem` при монтировании | Ошибка в `/etc/fstab` | Используйте `genfstab -U /mnt`. Для BTRFS убедитесь, что указано `subvol=@`. |
| Wayland не работает (NVIDIA) | Отключён KMS | Добавьте `nvidia-drm.modeset=1` в `GRUB_CMDLINE_LINUX_DEFAULT`, пересоберите GRUB и initramfs. |

### 📚 Путь самостоятельного изучения
Установка Arch Linux — это не финиш, а старт. Чтобы превратить этот проект в полноценный образовательный опыт:
1. 📖 Читайте [Arch Wiki](https://wiki.archlinux.org/) — это эталонный источник знаний.
2. 🔍 Используйте `man <команда>` и `journalctl -p 3 -xb` для анализа ошибок.
3. 🤝 Задавайте вопросы на [форуме Arch Linux](https://bbs.archlinux.org/) или в русскоязычных Telegram-чатах, прикладывая вывод `journalctl` и `lsblk`.
4. 🔄 Не бойтесь ломать систему. С `BTRFS + Snapper` вы всегда можете откатиться к рабочему состоянию через GRUB или Live-USB.
