# 🐧 Упрощённая установка Arch Linux: Макет блочной установки

## 📖 Введение
Установка Arch Linux может быть простой, быстрой и понятной при использовании специального **макета блочной установки**. Этот подход разработан специально для работы в **графической среде Live-образа** и предоставляет:
*   ✅ **Полный визуальный контроль**: вы видите весь макет установки перед глазами.
*   🖱️ **Удобство редактирования**: использование привычных графических редакторов (`mousepad`, `gedit`, `kate`) вместо консольных (nano/vim).
*   🚀 **Высокую скорость**: копирование готовых блоков команд целиком, а не ввод их вручную.
*   🛠️ **Гибкость**: легкая замена переменных (имя диска, пользователь) через функцию «Найти и заменить везде».

> 🛑 **КРИТИЧЕСКОЕ ПРЕДУПРЕЖДЕНИЕ О БЕЗОПАСНОСТИ**
>
> Все представленные макеты выполняют **ПОЛНУЮ ОЧИСТКУ ВЫБРАННОГО ДИСКА** (`wipefs`, `sgdisk -Z`).
> *   ⚠️ **Все данные на целевом диске будут безвозвратно удалены.**
> *   💾 Обязательно сделайте **резервную копию** всех важных данных перед началом.
> *   🔍 Тщательно проверьте имя диска (`sdx` / `nvme0n1`) перед выполнением команд разметки.
> *   Автор не несет ответственности за потерю данных в результате неверного следования инструкции.

## 🧭 Навигация по макетам

Выберите файл, соответствующий вашему оборудованию и требованиям безопасности:

| Файл макета                 | Описание                  | Требования         | Особенности                                         |
| :---                        | :---                      | :---               | :---                                                |
| **`Arch UEFI DRACUT.m`**           | Базовая установка (BTRFS) | UEFI, GPT          | Стандартная современная установка.                  |
| **`Arch BIOS DRACUT.m`**           | Базовая установка (BTRFS) | BIOS (Legacy), GPT | Требуется раздел BIOS Boot (EF02).                  |
| **`BTRFS LVM LUKS UEFI DRACUT.m`** | Полное шифрование + LVM   | UEFI, GPT          | Максимальная безопасность. Пароль при загрузке.     |
| **`BTRFS LVM LUKS BIOS DRACUT.m`** | Полное шифрование + LVM   | BIOS (Legacy), GPT | Требует отдельный незашифрованный `/boot`.          |
| **`README.md`**             | Эта инструкция            | -                  | Документация и чек-лист.                            |

> 💡 **Примечание:** Файлы с пометкой `LUKS` требуют ввода пароля расшифровки при каждой загрузке системы. Без этого пароля доступ к данным невозможен.

## 🚀 Способ установки: Графическая среда (Рекомендуется)

Первый метод предполагает загрузку в Live-образ с графическим интерфейсом (например, официальный ISO EndeavourOS, либо Garuda Linux использующие официальные репозитории Arch Linux).

Второй метод требует наличия двух компьютеров использующих SSH соединения между собой. \
Первый компьютер основной, он управляет через терминал всем процессом установки системы на втором компьютере, который загружен в Live-образ Arch Linux.

### Почему это удобно?
1.  **Графический редактор**: Вы открываете файл макета в удобном редакторе (`mousepad`, `gedit`, `kate`), где видно всю структуру сразу.
2.  **Поиск и замена везде**: Эта функция позволяет мгновенно заменить все переменные (имя диска, пользователя) во всем файле.
3.  **Копирование блоков**: Вы просто выделяете мышкой целый блок команд (например, «Блок 5: Форматирование») и вставляете его в терминал одним действием.
4.  **Визуальный контроль**: Вы видите комментарии к командам прямо в редакторе, что снижает риск ошибки.

### Пошаговый процесс:

1.  **Загрузка**: Загрузитесь с флешки в графическую среду Live-системы.
2.  **Терминал**: Откройте терминал.
3.  **Клонирование**:
    
    git clone https://github.com/rubleiura/arch-ru
    
4.  **Редактирование**:
    *   Откройте нужный файл (например, `Arch UEFI DRACUT.m`) в графическом редакторе:
        mousepad или используйте gedit, kate, xed в зависимости от вашей среды
    *   Выполните настройку переменных согласно инструкции ниже.
    *   **Сохраните файл**, используя меню редактора («Файл» → «Сохранить») или стандартную комбинацию клавиш.

5.  **Установка**:
    *   Разместите окна так, чтобы слева был редактор кода, а справа — терминал.
    *   Выделяйте мышкой целые блоки команд в редакторе.
    *   Копируйте и вставляйте (правой и левой кнопкой мыши) в терминал.
    *   Дождитесь выполнения блока перед переходом к следующему.

## 📋 Подготовка и настройка переменных

Перед запуском любых команд установки **критически важно** адаптировать макет под ваше оборудование.

### Шаг 1: Диагностика оборудования (Блок 2)
Запустите **Блок 2: ДИАГНОСТИКА ОБОРУДОВАНИЯ** из выбранного макета (скопируйте и вставьте в терминал). Изучите вывод:
*   [ ] Имя диска (например, `nvme0n1` или `sda`).
*   [ ] Тип процессора (Intel или AMD) — для выбора микрокода.
*   [ ] Объем ОЗУ — для расчета размера Swap.
*   [ ] Тип диска (SSD или HDD) — для параметров монтирования.

### Шаг 2: Редактирование переменных (Блок 3)
В открытом графическом редакторе найдите **Блок 3: НАСТРОЙКА ПЕРЕМЕННЫХ**. Замените значения-заглушки на ваши реальные данные.

| Переменная        | Описание                  | Пример значения          | Важное примечание                              |
| :---              | :---                      | :---                     | :---                                           |
| `sdx`             | Имя физического диска     | `nvme0n1` или `sda`      | Меняется **ДО** разметки (Этап 1)              |
| `sda1`, `sda2`... | Имена разделов            | `nvme0n1p1`, `nvme0n1p2` | Меняются **ПОСЛЕ** разметки (Этап 2)*          |
| `forename`        | Имя пользователя (login)  | `yuri`                   | Только латиница, без пробелов                  |
| `User Name`       | Полное имя                | `Yuri Rublev`            | Можно кириллицу, с пробелами                   |
| `Sony`            | Имя компьютера (Hostname) | `arch-pc`                | Без пробелов                                   |
| `amd-ucode`       | Микрокод CPU              | `intel-ucode`            | Заменить на `intel-ucode` если процессор Intel |
| `4G`              | Размер Swap               | `8G` или `16G`           | Для гибернации ≥ объема ОЗУ                    |
| `defaults`        | Опции монтирования        | `ssd,noatime...`         | Используйте рекомендации из Блока 2            |

> ⚠️ **ОСОБЕННОСТИ ЗАМЕНЫ РАЗДЕЛОВ (ЭТАП 2)**
>
> Структура разделов отличается в зависимости от макета! Будьте предельно внимательны при замене `sda1`, `sda2`, `sda3`:
>
> **1. Обычные макеты (`Arch UEFI DRACUT.m`, `Arch BIOS DRACUT.m`):**
> *   `sda1` → EFI раздел (UEFI) или BIOS Boot (Legacy).
> *   `sda2` → Корневой раздел (`/`).
> *   `sda3` → Swap.
>
> **2. Макеты с шифрованием UEFI (`BTRFS LVM LUKS UEFI DRACUT.m`):**
> *   `sda1` → EFI раздел (ESP).
> *   `sda2` → Зашифрованный контейнер (внутри него LVM).
>
> **3. Макеты с шифрованием BIOS (`BTRFS LVM LUKS BIOS DRACUT.m`):**
> *   `sda1` → **BIOS Boot Partition** (не форматируется, не монтируется!).
> *   `sda2` → Раздел `/boot` (ext4, **не шифруется**).
> *   `sda3` → Зашифрованный контейнер (внутри него LVM).
>
> **Всегда сверяйтесь с таблицей переменных внутри конкретного файла `.m` перед заменой!**

### Шаг 3: Глобальная замена (Find & Replace)
Используйте функцию «Найти и заменить везде» в графическом редакторе:
*   [ ] Найти: `sdx`      → Заменить на: `ваш_диск` (во всем файле)
*   [ ] Найти: `forename` → Заменить на: `ваш_юзер`
*   [ ] Найти: `Sony`     → Заменить на: `ваш_hostname`
*   [ ] ...и так далее для всех переменных из таблицы.
*   *Примечание:* Не заменяйте имена разделов (`sda1`, `sda2`) до выполнения Блока 4 (разметки)!

### Шаг 4: Финальная проверка
*   [ ] Пробежитесь глазами по файлу. Убедитесь, что нет остатков старых имен.
*   [ ] **Сохраните файл** средствами вашего текстового редактора.

## 🖥️ Установка графических сред и драйверов

### Видеодрайверы
Инструкции по установке драйверов (Intel, AMD, NVIDIA) уже встроены в **Блок 12** каждого основного макета.
*   Не нужно искать отдельные файлы гайдов.
*   Выберите секцию, соответствующую вашей видеокарте, внутри Блока 12.
*   ⚠️ **Важно:** Выполняйте эти команды только находясь внутри среды `chroot` (`arch-chroot /mnt`).
*   Для карт NVIDIA обязательно выполните команды настройки модулей и пересборки `initramfs` в конце блока.
*   *Примечание:* В блоках установки GNOME в файлах `.m` могут встречаться опечатки (лишние пробелы в названиях пакетов, например `file-rol ler`). При копировании команд из файла `.m` внимательно проверяйте названия пакетов или используйте исправленные команды из официальной документации, если возникнет ошибка.

### Графические окружения (DE)
После установки базы и драйверов перейдите к **Блоку 14**. Там представлены готовые команды для установки:
*   KDE Plasma
*   GNOME
*   XFCE4
*   MATE
*   Cinnamon
*   LXQt / LXDE

Просто скопируйте блок нужного окружения и вставьте в терминал.

## 🏁 ФИНАЛЬНЫЕ ЭТАПЫ: Завершение и настройка (Критически важно!)

После установки графической среды (Блок 14) установка еще не завершена полностью. Необходимо корректно выйти из среды установки и настроить инструменты для работы с Btrfs.

**Внимательно следуйте инструкциям внутри вашего файла `.m`:**

### 🛑 ШАГ 1: Выполните БЛОК 15 (Завершение установки)
Находится в самом низу файла макета (сразу после блоков DE).
1.  Откройте ваш файл `.m` в редакторе.
2.  Найдите раздел **`🧹 [LIVE] БЛОК 15: ЗАВЕРШЕНИЕ УСТАНОВКИ`**.
3.  Скопируйте и выполните команды этого блока в терминал (находясь всё ещё в Live-среде).
    *   Этот блок выполнит размонтирование разделов (`umount`), отключение swap и выключение компьютера (`poweroff`).
4.  Извлеките установочную флешку.
5.  Включите компьютер и загрузитесь в вашу новую систему Arch Linux.
6.  Войдите под своим пользователем (`forename`).

### 🚀 ШАГ 2: Выполните БЛОК 16 (Настройка после первой загрузки)
**⚠️ ВАЖНО:** Этот блок выполняется ТОЛЬКО после того, как вы загрузились в установленную систему (не в Live-USB!).
1.  Откройте терминал в вашей новой системе.
2.  Снова откройте файл макета (`.m`) в редакторе.
3.  Найдите раздел **`📋 [USER] БЛОК 16: РЕКОМЕНДАЦИИ ПОСЛЕ УСТАНОВКИ`**.
4.  Последовательно скопируйте и выполните команды из этого блока:
    *   **Установка Помошника AUR: `yay`.
    *   **Установка утилит Btrfs: `snapper`, `btrfs-assistant` и необходимые поддержки.
    *   **Активация таймера:** Включит автоматическое создание временных снапшотов.
5.  **Настройка Btrfs Assistant:**
    *   Запустите **Btrfs Assistant** из меню приложений.
    *   Программа может запросить пароль администратора (sudo).
    *   Теперь вы можете создавать снапшоты, настраивать автоматическое сохранение перед обновлением и выполнять откат системы (rollback) через меню GRUB в случае сбоя.

> ✅ **Поздравляем!** Ваша система Arch Linux полностью установлена, настроена и готова к работе с продвинутыми функциями Btrfs благодаря выполнению Блоков 15 и 16.

## 📢 Критические рекомендации и отладка

### 🔍 Контроль процесса
1.  **Копируйте блоками или целыми командами целиком.** .
2.  **Следите за выводом.** Если команда выдала ошибку — остановитесь, проанализируйте сообщение и устраните причину перед переходом к следующему блоку.
3.  **Переменные.** Двойная проверка замененных переменных (`sdx`, имена разделов) спасет ваши данные.

### 🆘 Частые проблемы и решения
*   **"Система не загружается, зависает на черном экране."**
    *   Проверьте, что вы выполнили шаг по настройке `mkinitcpio.conf` и команду `mkinitcpio -P` в Блоке 10 или 12.
    *   Убедитесь, что в `HOOKS` присутствуют `encrypt`, `lvm2` (для версий с шифрованием) в правильном порядке.
*   **"Ошибка при монтировании разделов (unrecognized FS)."**
    *   Проверьте файл `/etc/fstab`. Убедитесь, что вы используете `UUID=...` вместо `/dev/sdXN` и что тип файловой системы указан верно.
    *   Для BTRFS проверьте параметр `subvol=@`.
*   **"Загрузчик GRUB не появляется."**
    *   Если вы устанавливаете в UEFI-режиме, убедитесь, что выполнили `grub-install` с флагами `--target=x86_64-efi` и `--efi-directory=/boot` (или `/boot/efi` в зависимости от макета).
    *   Для BIOS+GPT убедитесь, что создан раздел BIOS Boot (EF02).
