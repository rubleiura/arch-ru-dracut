# 🐧 Упрощённая установка Arch Linux: Макет блочной установки

## 📖 Введение

Установка Arch Linux может быть простой, быстрой и понятной при использовании специального макета блочной установки. Этот подход разработан специально для работы в графической среде Live-образа и предоставляет:

- ✅ **Полный визуальный контроль:** вы видите весь макет установки перед глазами.
- 🖱️ **Удобство редактирования:** использование привычных графических редакторов (`mousepad`, `gedit`, `kate`) вместо консольных (`nano`/`vim`).
- 🚀 **Высокую скорость:** копирование готовых блоков команд целиком, а не ввод их вручную.
- 🛠️ **Гибкость:** легкая замена переменных (имя диска, пользователь) через функцию «Найти и заменить везде».

## 🚀 Способ установки: Графическая среда (Рекомендуется)

Первый метод предполагает загрузку в Live-образ с графическим интерфейсом (например, официальный ISO EndeavourOS, либо Garuda Linux, использующие официальные репозитории Arch Linux).

Второй метод требует наличия двух компьютеров, использующих SSH-соединения между собой. Первый компьютер (основной) управляет через терминал всем процессом установки системы на втором компьютере, который загружен в Live-образ Arch Linux.

### Почему это удобно?

- **Графический редактор:** Вы открываете файл макета в удобном редакторе (`mousepad`, `gedit`, `kate`), где видно всю структуру сразу.
- **Поиск и замена везде:** Эта функция позволяет мгновенно заменить все переменные (имя диска, пользователя) во всём файле.
- **Копирование блоков:** Вы просто выделяете мышкой целый блок команд (например, «Блок 5: Форматирование») и вставляете его в терминал одним действием.
- **Визуальный контроль:** Вы видите комментарии к командам прямо в редакторе, что снижает риск ошибки.

### Пошаговый процесс:

1. **Загрузка:** Загрузитесь с флешки в графическую среду Live-системы.
2. **Терминал:** Откройте терминал.
3. **Клонирование:**
   ```bash
   git clone https://github.com/rubleiura/arch-ru-dracut
   ```
4. **Редактирование:** Откройте нужный файл (например, `Arch UEFI DRACUT.m`) в графическом редакторе. Выполните настройку переменных согласно инструкции ниже. Сохраните файл.
5. **Установка:** Разместите окна так, чтобы слева был редактор кода, а справа — терминал. Выделяйте мышкой целые блоки команд в редакторе, копируйте и вставляйте в терминал. Дождитесь выполнения блока перед переходом к следующему.

## 📋 Подготовка и настройка переменных

Перед запуском любых команд установки критически важно адаптировать макет под ваше оборудование.

### 🔹 Шаг 1: Динамический подбор зеркал (Блок 0)

В начале каждого макета находится Python-скрипт `light_countries.py`. Он определяет ваше местоположение по IP и выводит 10 ближайших стран.

✅ **Действие:** Запустите скрипт → скопируйте полученный список → замените слово `Russia` в конфигурации `reflector` (Блоки 1 и 7) на этот список. Это гарантирует максимальную скорость загрузки пакетов.

### 🔹 Шаг 2: Диагностика оборудования (Блок 2)

Запустите **Блок 2: ДИАГНОСТИКА ОБОРУДОВАНИЯ** из выбранного макета. Изучите вывод:

- [ ] Имя диска (например, `nvme0n1` или `sda`).
- [ ] Тип процессора (Intel или AMD) — для выбора микрокода.
- [ ] Объем ОЗУ — для расчета размера Swap.
- [ ] Тип диска (SSD или HDD) — для параметров монтирования.

### 🔹 Шаг 3: Редактирование переменных (Блок 3)

> ⚠️ **ВАЖНО:** Замена переменных происходит в **ДВА ЭТАПА!** Нарушение порядка приведет к ошибке разметки.

- **ЭТАП 1 (СЕЙЧАС, до Блока 4):** Замените `sdx`, размер SWAP, `Sony`, `forename`, `User Name`, `amd-ucode`, `linux-lts`, `defaults`.
- **ЭТАП 2 (ПОСЛЕ Блока 4 — разметки диска):** Используйте вывод `lsblk` из Блока 4 и замените шаблоны разделов (`sda1`, `sda2`, `sda3`) на реальные имена ваших разделов.

| Переменная        | Описание                  | Пример значения     | Когда менять?          |
|-------------------|---------------------------|---------------------|------------------------|
| `sdx`             | Имя физического диска     | `nvme0n1` или `sda` | ЭТАП 1 (сейчас)        |
| `sda1`, `sda2`... | Имена созданных разделов  | `nvme0n1p1`...      | ЭТАП 2 (после Блока 4) |
| `forename`        | Имя пользователя (login)  | `yuri`              | ЭТАП 1 (сейчас)        |
| `User Name`       | Полное имя                | `Yuri Rublev`       | ЭТАП 1 (сейчас)        |
| `Sony`            | Имя компьютера (Hostname) | `arch-pc`           | ЭТАП 1 (сейчас)        |
| `amd-ucode`       | Микрокод CPU              | `intel-ucode`       | ЭТАП 1 (сейчас)        |
| `4G`              | Размер Swap               | `8G` или `16G`      | ЭТАП 1 (сейчас)        |
| `defaults`        | Опции монтирования BTRFS  | `ssd,noatime...`    | ЭТАП 1 (сейчас)        |

### 🔹 Структура разделов по конфигурациям

Будьте предельно внимательны при замене имён разделов на **ЭТАПЕ 2**:

1. **`Arch UEFI DRACUT.m`** (UEFI, без шифрования):
   - `sda1` → ESP (FAT32, `/boot`)
   - `sda2` → Корень (`/`, BTRFS)
   - `sda3` → Swap

2. **`Arch BIOS DRACUT.m`** (BIOS+GPT, без шифрования):
   - `sda1` → BIOS Boot Partition (EF02, **НЕ форматируется!**)
   - `sda2` → Корень (`/`, BTRFS)
   - `sda3` → Swap

3. **`BTRFS LVM LUKS UEFI DRACUT.m`** (UEFI + шифрование):
   - `sda1` → ESP (FAT32, `/boot`)
   - `sda2` → Зашифрованный контейнер (внутри LVM + BTRFS)
   - *Swap создаётся внутри LVM, отдельного раздела нет.*

4. **`BTRFS LVM LUKS BIOS DRACUT.m`** (BIOS+GPT + шифрование):
   - `sda1` → BIOS Boot Partition (EF02, **НЕ форматируется!**)
   - `sda2` → Незашифрованный `/boot` (ext4, 1 ГБ)
   - `sda3` → Зашифрованный контейнер (внутри LVM + BTRFS)

### 🔹 Шаг 4: Глобальная замена (Find & Replace)

Используйте функцию «Найти и заменить везде» в графическом редакторе:

- [ ] Найти: `sdx` → Заменить на: `ваш_диск` (во всём файле)
- [ ] Найти: `forename` → Заменить на: `ваш_юзер`
- [ ] Найти: `Sony` → Заменить на: `ваш_hostname`
- [ ] ...и так далее для всех переменных из таблицы.

> 📌 **Примечание:** Никогда не заменяйте имена разделов (`sda1`, `sda2`) до выполнения Блока 4!

### 🔹 Шаг 5: Финальная проверка

- [ ] Пробежитесь глазами по файлу. Убедитесь, что нет остатков старых имён.
- [ ] Сохраните файл средствами вашего текстового редактора.

## 📢 Критические рекомендации и отладка

### 🔍 Контроль процесса

- Копируйте блоками или целыми командами целиком.
- Следите за выводом. Если команда выдала ошибку — остановитесь, проанализируйте сообщение и устраните причину перед переходом к следующему блоку.
- **Переменные.** Двойная проверка замененных переменных (`sdx`, имена разделов) спасет ваши данные.

### 🆘 Частые проблемы и решения

**«Система не загружается, зависает на чёрном экране или падает в dracut shell.»**

- Убедитесь, что хуки `mkinitcpio` отключены (сделано автоматически в Блоке 10 через symlink на `/dev/null`).
- Проверьте параметры ядра в `grub.cfg`: наличие `rd.luks.uuid=...`, `rd.lvm.lv=...`, `resume=UUID=...`.
- Убедитесь, что образ пересобран: `dracut --force /boot/initramfs-linux-lts.img`.
- Проверьте конфигурацию: `cat /etc/dracut.conf.d/custom.conf`.

**«Ошибка при монтировании разделов или загрузке BTRFS.»**

- Проверьте `/etc/fstab`. Убедитесь, что используется `UUID=...` и параметр `subvol=@`.
- В LUKS+BIOS конфигурации `/boot` должен быть `ext4` и монтироваться в `/boot`, а не `/boot/efi`.

**«GRUB не появляется или не видит другие ОС.»**

- **UEFI:** проверьте `grub-install --target=x86_64-efi --efi-directory=/boot`.
- **BIOS:** проверьте `grub-install --target=i386-pc /dev/sdx` (установка в MBR диска, а не раздела).
- Убедитесь, что `os-prober` включён, а `GRUB_DISABLE_RECOVERY=true` задан в `/etc/default/grub`.
